# NovelTrans Architecture Notes

## Build and Release

`src/` is the source of truth. `dist/` is generated by `npm run build` or `npm test` and is ignored by git.

Release packages are published through the GitHub `Release` workflow. The workflow installs dependencies, runs tests, and publishes with `npm publish --provenance`; `prepack` runs the production TypeScript build so packaged artifacts contain `dist/`.

Use `npm run pack:check` before a release to inspect the package contents.

## Runtime Flow

Import creates a project directory with source episodes, project metadata, a glossary, SQLite episode state, and logs. Translation workers pull pending episode state, call the configured adapter, persist JSON and Markdown translations, run QA, merge glossary candidates, and refresh quality reports. Export reads persisted translations and writes TXT/EPUB outputs.

The shared episode lifecycle lives in `src/engine/episodeLifecycle.ts`. CLI queue translation, TUI sessions, and review-desk single-episode retranslation should reuse it for translate/persist/QA/glossary behavior.

## UI Boundary

`src/ui-v2/` is the active terminal app. `src/ui/` still contains data helpers and action helpers consumed by the v2 app, so it is not dead code. New interactive screens should go under `src/ui-v2/`; reusable project actions may stay in `src/ui/actions/` when they are UI-version neutral.

## Storage Compatibility

New episode artifacts use 5-digit padding, for example `episode_00001.json`, so lexicographic filenames remain stable for long serials. The storage layer still reads and overwrites existing 3-digit artifacts such as `episode_001.json` for old projects.

## SQLite Warning

Project state uses Node's built-in `node:sqlite` `DatabaseSync`, which is still experimental in Node 22. The runtime suppresses that specific warning while loading the module. Keep this dependency isolated in `src/storage/stateStore.ts` so future Node API changes are localized.

## Credentials

The local credential file is protected primarily by `0600` filesystem permissions. The stored value is not intended to provide strong confidentiality against a user or process that can read the config directory. Do not describe it as OS-keychain-equivalent encryption.
